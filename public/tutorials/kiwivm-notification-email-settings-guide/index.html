搬瓦工KiwiVM控制面板邮件通知设置指南

使用搬瓦工VPS的朋友都知道，KiwiVM控制面板会发送各种邮件通知，从流量使用提醒到系统维护通知，应有尽有。但问题是，邮件太多了！

邮箱里每天都有好几封搬瓦工的邮件，有些是重要的流量警告，有些只是普通的系统信息，结果重要的通知反而容易被埋没在众多邮件中。

其实，KiwiVM控制面板提供了非常灵活的邮件通知管理功能，你可以精确控制哪些通知需要发邮件，哪些不需要。今天我就来详细介绍这个功能，帮你找到最适合的通知设置。

KiwiVM邮件通知设置界面#

要进入邮件通知设置页面，你需要先登录 搬瓦工官网 ，打开KiwiVM控制面板，然后在左侧菜单中找到"Email Notifications"选项。

如上图所示，通知设置界面将所有邮件通知分为四大类：带宽使用通知、安全通知、系统通知和其他通知。每一类通知都可以独立开启或关闭，非常方便。

界面设计很简洁，每个通知类型前面都有一个复选框，勾选表示开启该类邮件通知，取消勾选则关闭。修改设置后，记得保存配置。

各类通知详解及建议#

流量使用通知 (Bandwidth Alerts)#

这是我认为最重要的通知类型，包括以下选项：

  • Bandwidth usage alert at 40% threshold
  • Bandwidth usage alert at 50% threshold
  • Bandwidth usage alert at 60% threshold
  • Bandwidth usage alert at 70% threshold
  • Bandwidth usage alert at 80% threshold
  • Bandwidth usage alert at 90% threshold
  • Bandwidth usage alert at 95% threshold
  • Bandwidth usage alert at 98% threshold
  • Bandwidth usage alert at 100% threshold

我的建议：

  • 必须开启：80%、90%、95%、100%
  • 可选开启：70%（如果你经常接近流量限制）
  • 不建议开启：40%、50%、60%（太早期的提醒，意义不大）

理由： 80%是一个重要的预警点，此时你还有时间考虑是否需要升级套餐或者减少使用量。90%和95%是紧急警告，100%则意味着服务即将或已经暂停，这些都是必须知道的信息。

安全通知 (Security Notifications)#

安全相关的通知包括：

  • Successful login to KiwiVM
  • Two-factor authentication disabled
  • Two-factor authentication enabled
  • Login with two-factor authentication backup key
  • KiwiVM password changed
  • KiwiVM password reset

我的建议：

  • 必须开启：KiwiVM password changed, KiwiVM password reset
  • 建议开启：Two-factor authentication disabled, Two-factor authentication enabled
  • 可选开启：Successful login to KiwiVM（如果你担心账户安全）
  • 不建议开启：Login with two-factor authentication backup key（除非你经常使用备用密钥）

理由： 密码修改和重置是最关键的安全事件，必须及时知道。二次验证的启用和禁用也很重要，因为这直接影响账户安全等级。

系统通知 (System Notifications)#

系统相关的通知包括：

  • VM migration completed
  • OS reinstall completed
  • OS reinstall completed with custom SSH keys
  • Snapshot created
  • Snapshot restored
  • Snapshot imported from another VM
  • Snapshot imported from Backup Vault

我的建议：

  • 建议开启：VM migration completed, OS reinstall completed
  • 按需开启：其他快照相关通知（如果你经常使用快照功能）

理由： VM迁移和系统重装是重要的系统操作，完成后的通知能让你及时了解状态。快照相关的通知则根据使用频率来决定是否开启。

其他通知 (Miscellaneous)#

这一类包括：

  • IP address replaced
  • Inbound DDoS notification

我的建议：

  • 必须开启：Inbound DDoS notification
  • 建议开启：IP address replaced

理由： DDoS攻击通知非常重要，因为严重的攻击可能导致IP被封禁或者服务中断。IP地址更换通知也很有用，特别是当你的服务依赖于特定IP时。

常见问题解答#

Q1：设置保存后多久生效？#

+
通知设置修改后立即生效，无需等待。如果你担心设置是否成功，可以故意触发一个通知来测试（比如创建一个快照）。

Q2：为什么我收不到任何邮件通知？#

+
首先检查垃圾邮件文件夹，很多邮件服务商会将自动化通知邮件标记为垃圾邮件。如果问题持续存在，建议查看我们的收不到搬瓦工邮件的解决方案文章，里面有详细的排查方法。

Q3：可以更改接收通知的邮箱地址吗？#

+
通知邮件会发送到你注册搬瓦工账户的邮箱，如果要更改接收地址，需要在账户设置中修改邮箱地址。

Q4：关闭某个通知后，还能在控制面板中看到相关信息吗？#

+
可以的，关闭邮件通知只是不发送邮件，控制面板中的信息和日志仍然会正常显示。

Q5：建议多久检查一次通知设置？#

+
一般来说，设置好后就不需要频繁修改。但如果你的使用模式发生变化（比如从个人使用转为商业使用），建议重新评估通知设置。
