前段时间，我陆续完成了搬瓦工美西几个 Basic VPS 机房的测试，包括洛杉矶 DC2 以及弗里蒙特机房的详细评测。今天，我带来了搬瓦工在美东地区的表现。

今天我要和你们分享的是搬瓦工纽约 USNY_6 机房 Basic VPS 的完整测试结果。相比美西机房，纽约机房位于美国东部，到国内的网络距离更远，理论上延迟会更高一些。这次测试我会从 CPU 性能、网络速度、延迟表现、硬盘 I/O、稳定性等多个维度来全面评估这台 VPS 的表现。

需要特别提醒的是，VPS的性能会受到很多因素影响，包括测试时间、网络状况、服务器负载等。我的测试结果仅供参考，你们在实际使用中可能会有不同的体验。

CPU 表现总结#

从测试结果来看，这台纽约 USNY_6 机房的 VPS 在 CPU 性能方面表现相当不错。单核 828.40 的事件处理能力在同配置产品中属于中上水平，而双核近乎完美的扩展性更是让人印象深刻。

这个性能表现还比我之前测试的洛杉矶 DC8 机房还要好。这可能得益于 USNY_6 机房采用了更新的硬件配置，Intel Xeon SierraForest 架构的处理器在单核性能和多核协作方面都有不错的提升。能够轻松胜任日常的网站运行、数据处理等任务。

不过需要注意的是，VPS 的 CPU 性能会受到邻居效应的影响。如果同一物理服务器上的其他虚拟机占用资源较多，你的性能可能会有所下降。从目前的测试结果来看，服务器的整体负载应该还比较合理。

网络性能总结#

综合各项网络测试的结果，这台纽约 USNY_6 机房的 VPS 在网络性能方面表现中规中矩。本地带宽充足，基本不存在瓶颈问题。国内延迟虽然不算特别低，但考虑到地理距离因素，这个表现是可以接受的。

对于一般的网站托管、个人博客等应用来说，这样的网络性能完全够用。但如果是对实时性要求较高的应用，比如在线游戏、视频会议等，可能会感觉到一定的延迟。
